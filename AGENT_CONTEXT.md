## 5. SECURITY & DATA PRACTICES

### Security Posture
1. **Only the Right People See the Right Data** — Access is scoped, reviewed, and time-bound with minimum needed privileges
2. **Your Data is Locked Up Tight** — AES (128/192/256) at rest; TLS 1.2+ in transit with TLS 1.3 roadmap
3. **We Only Use What We Need** — Collect/process only what's needed; prefer tokenization/redaction
4. **Our Code is Tested for Safety** — Build & test against OWASP ASVS controls with shift-left security in CI
5. **Everything is Tracked and Recorded** — Comprehensive audit trails, access logs, and break-glass procedures
6. **We Follow Industry Standards** — SOC 2 Trust Services Criteria alignment

### Data Handling
- **PII:** Default to minimization, pseudonymize where feasible, manage secrets via KMS/Secret Manager, rotate keys
- **Encryption:** AES at rest, TLS 1.2+ in transit, migration to TLS 1.3, FIPS-compliant cipher suites
- **Residency:** Per-customer isolation, BYO-VPC option, data residency options, NIST CSF alignment

### How DOTS.ai Works With Your Data
- DOTS.ai runs OUTSIDE your systems, talking only through APIs/webhooks
- No sensitive data leaves your perimeter
- Every agent operates with read-only signals and transmits metadata-only summaries
- Can work with summary data, anonymized datasets, or only via APIs without storing anything sensitive
- Does NOT replace existing tools — augments them via middleware agents

### Identity & Access
- Role-based access with auto-expiring credentials

### Reliability & Operations
