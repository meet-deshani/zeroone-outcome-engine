# Access Level Matrix

> **Access:** TEAM

---

## Purpose

This document defines which documentation is appropriate for which audience. Use this matrix when sharing docs with investors, team members, or community members.

---

## Access Levels Defined

| Level | Audience | What They Should See | What They Should NOT See |
|-------|----------|---------------------|------------------------|
| **INVESTOR** | Investors, advisors, board members | Vision, traction, competitive positioning, architecture overview, security posture, IP ownership | Internal IPs, credentials, server configs, operational details |
| **TEAM** | Internal team, engineering partners | Everything — full technical depth, infrastructure details, operational guides | Nothing restricted (highest access level) |
| **COMMUNITY** | Prospects, community members, public | Services, pricing, how to engage, learning paths, bounty engine | Internal architecture, server details, traction metrics |

## Sharing Protocols

1. **Never share raw files** — always share as PDF exports or read-only links
2. **Watermark investor docs** — add recipient name to shared investor documents
3. **Review before sharing** — always re-read the document before sharing to ensure no sensitive data crept in
4. **Track who has what** — maintain a log of which documents were shared with whom
5. **Update regularly** — docs should be refreshed quarterly to reflect current state
